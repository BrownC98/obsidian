만든게임 개요
2048 게임의 변형으로 플레이어는 빈 칸을 클릭하여 타일을 배치하고 시스템은 배치된 타일들을 이동시켜 최대한 많은 점수를 얻어내는 것이 목적, 시스템이 획득한 점수만큼 플레이어의 HP 가 감소함, 플레이어는 게임이 시작할 떄 1000의 체력을 가짐

게임 클리어 조건
플레이어가 빈칸을 모두 타일로 채우고 더이상 시스템이 타일이동을 할 수 없으면 플레이어의 승리, HP가 0이되면 플레이어의 패배다.

턴 시스템
플레이어가 타일을 배치할 순서가 되면 6초간의 제한시간이 주어지고 제한시간 안에 타일을 배치하지 않으면 해당 턴은 방치된 턴으로 카운트 된다.

방치 보상 시스템
연속으로 방치된 턴이 3, 5, 7 턴이 되면 '방치 보상 선택'기회가 주어진다. 해당 선택에서는 게임플레이에 도움을 주는 효과들 중 하나를 선택할 수 있다. 각 선택기회에는 3가지 선택지가 주어진다. 방치한 턴이 길어질 수록 보상도 강력해진다. 각 선택지는 발동할 효과를 선택하거나, 지금 선택하지 않고 턴을 더 방치하는 선택지가 주어진다.

스킬 시스템
각 게임이 시작하기 전에 플레이어가 사용할 수 있는 스킬을 6개중 하나를 선택할 수 있다. 각 스킬은 고유한 기능을 가지고 있으며 각각의 턴제 쿨타임을 가지고 있다.


QA가 게임의 완성을 어떤 기준으로 결정하는지 알아야함

게임 QA 프로세스 단계는 다음과 같이 구체적으로 진행됩니다.

1. **계획 수립 (Planning)**
    
    - **테스트 목표 및 범위 설정:** 게임의 주요 기능(예: 타일 이동, 방치 보상, 스킬 효과 등)과 사용자 경험 전반을 검증할 영역을 명확히 정합니다.
    - **일정 및 리소스 배정:** 전체 테스트 일정, 필요한 인력, 하드웨어 및 소프트웨어 환경 등 자원을 계획에 반영하여 효율적으로 분배합니다.
    - **주요 테스트 케이스 설계:** 초기 테스트 목표를 기반으로 게임의 각 기능에 맞는 테스트 시나리오와 케이스를 설계하며, 이를 통해 테스트의 기본 틀을 마련합니다.
2. **테스트 케이스 개발 (Test Case Development)**
    
    - **세분화된 테스트 시나리오 작성:** 기능별, 시나리오 기반, 그리고 예상치 못한 엣지 케이스까지 모두 포함할 수 있도록 상세하게 작성합니다.
    - **명확한 절차 및 기대 결과 설정:** 각 테스트 케이스에는 실행 단계, 전제 조건, 예상 결과 등을 구체적으로 명시하여 누구나 재현 가능한 형태로 만듭니다.
    - **자동화 도구와의 연계:** 반복적인 테스트는 자동화 스크립트를 작성하여 효율성을 높이는 것도 포함됩니다.
3. **테스트 실행 (Execution)**
    
    - **실제 플레이 환경 적용:** 실제 게임 환경과 유사한 조건에서 테스트 케이스를 실행해 게임의 정상 작동 여부를 확인합니다.
    - **버그 및 문제점 기록:** 테스트 중 발생하는 오류, 예기치 않은 행동, 성능 저하 등을 상세히 기록하며, 필요한 경우 스크린샷, 로그, 영상 등을 함께 첨부합니다.
    - **다양한 시나리오 검증:** 플레이어가 직접 체험할 다양한 상황을 모의하여, 일반 사용자뿐만 아니라 극한 상황에서도 게임이 어떻게 작동하는지 확인합니다.
4. **결과 분석 및 보고 (Analysis & Reporting)**
    
    - **문제 우선순위 결정:** 수집된 데이터를 바탕으로 버그의 심각도와 발생 빈도를 분석하여, 수정의 우선순위를 정합니다.
    - **상세 보고서 작성:** 재현 절차, 오류 메시지, 영향을 받은 기능 등을 포함한 보고서를 작성하여 개발팀과 공유합니다.
    - **피드백 반영:** 보고된 문제를 바탕으로 개발팀이 수정한 후, 수정 사항이 올바르게 반영되었는지 다시 확인할 수 있도록 합니다.
5. **회귀 테스트 (Regression Testing)**
    
    - **수정사항 검증:** 개발팀에서 버그를 수정한 후, 기존 기능에 영향이 없는지 전체 시스템을 다시 테스트합니다.
    - **지속적인 품질 보증:** 이전에 통과했던 테스트 케이스들을 재실행하여, 새로운 코드 변경이 다른 기능에 부정적 영향을 미치지 않았는지 확인합니다.
    - **테스트 자동화 활용:** 반복적인 회귀 테스트를 자동화하여, 수정 후 빠르고 정확하게 검증할 수 있도록 합니다.

## 1. 테스트 목표 및 범위 설정

### **가. 기능성 검증**

- **타일 이동 및 배치:**
    
    - 플레이어가 빈 칸을 클릭하여 타일을 배치할 수 있는지
    - 시스템이 배치된 타일을 올바른 방향으로 이동시키고, 2048 게임의 병합 로직(같은 숫자 타일의 병합 등)이 정확하게 작동하는지
- **HP 관리 시스템:**
    
    - 시스템의 타일 이동 점수에 따라 플레이어의 HP(초기 1000)가 정확하게 감소하는지
    - HP 감소 계산 로직과 임계치(HP 0) 도달 시 패배 조건이 올바르게 적용되는지
- **방치 보상 시스템:**
    
    - 6초 내에 타일을 배치하지 않은 경우, 해당 턴이 ‘방치’로 기록되는지
    - 연속 방치 턴(3, 5, 7턴)마다 보상 선택 기회가 정상적으로 제공되는지
    - 보상 선택 시 제시되는 옵션들이 실제 게임플레이에 도움이 되는 효과를 올바르게 발동하는지
- **스킬 시스템:**
    
    - 게임 시작 시 6개 스킬 중 한 가지를 선택할 수 있는 인터페이스 제공
    - 각 스킬의 고유 효과와 턴제 쿨타임이 정상적으로 작동하는지
    - 스킬 사용 시 게임 상태(타일 이동, 점수 산출, 보너스 효과 등)에 올바른 영향을 미치는지

### **나. 사용자 경험(UX) 검증**

- **UI/UX 요소:**
    
    - 각 기능(타일 배치, 보상 선택, 스킬 선택 등)의 인터페이스가 직관적이고 반응속도가 빠른지
    - 애니메이션 및 효과(타일 이동, 병합, HP 감소, 보상 발동 등)가 부드럽게 진행되는지
- **턴 시스템:**
    
    - 6초 제한 타이머가 정확하게 작동하며, 제한 시간 내 입력이 없을 경우 자동으로 방치 처리되는지
    - 플레이어 입력과 시스템 반응 간의 타이밍이 적절하게 동기화되는지

## 2. 주요 테스트 케이스 설계

### **가. 타일 이동 및 배치 관련 테스트 케이스**

- **TC-01: 정상 배치 및 이동**
    
    - **목표:** 플레이어가 빈 칸을 클릭하여 타일을 배치하면, 시스템이 해당 타일을 지정된 방향으로 올바르게 이동시키는지 확인
    - **절차:**
        1. 빈칸 클릭 후 타일 배치
        2. 시스템이 타일 이동을 시작하는지 관찰
        3. 이동 경로와 최종 위치가 규칙에 맞게 결정되는지 확인
    - **예상 결과:** 타일이 올바른 방향으로 이동하며 병합 조건(같은 숫자일 경우)이 충족되면 병합됨
- **TC-02: 병합 로직 검증**
    
    - **목표:** 인접한 동일 숫자 타일의 병합이 정확히 처리되는지 확인
    - **절차:**
        1. 동일 숫자 타일을 인접하게 배치
        2. 시스템 이동 로직 실행
        3. 병합 후 생성된 타일의 숫자와 위치 확인
    - **예상 결과:** 두 타일이 병합되어 숫자가 두 배로 증가한 타일로 변환됨
- **TC-03: 예외 상황 처리 (동시 다중 입력 등)**
    
    - **목표:** 플레이어가 짧은 시간 내 연속해서 입력할 경우, 시스템이 안정적으로 처리하는지 확인
    - **절차:**
        1. 빠른 연속 클릭 및 배치 시도
        2. 시스템의 타일 이동 및 병합 처리 관찰
    - **예상 결과:** 예외 상황에서도 시스템이 충돌 없이 정상 작동함

### **나. 방치 보상 시스템 관련 테스트 케이스**

- **TC-04: 6초 미입력 시 방치 처리**
    
    - **목표:** 6초 이내 입력이 없을 경우 해당 턴이 방치로 기록되는지 확인
    - **절차:**
        1. 타일 배치 입력 없이 6초 경과
        2. 방치 턴으로 기록되는지 시스템 로그 및 UI 확인
    - **예상 결과:** 해당 턴이 방치 처리되고, 연속 방치 횟수가 1 증가함
- **TC-05: 연속 방치 시 보상 선택 기회 제공**
    
    - **목표:** 연속 방치 턴이 3, 5, 7턴 도달 시 보상 선택 UI가 나타나는지 확인
    - **절차:**
        1. 연속 방치 턴 3회 달성 후 보상 선택 옵션 등장 여부 확인
        2. 동일하게 5, 7턴 달성 시에도 보상 선택 옵션 확인
    - **예상 결과:** 각 조건 충족 시, 사전에 정의된 보상 선택 옵션이 정상적으로 노출됨
- **TC-06: 보상 선택 효과 검증**
    
    - **목표:** 보상 선택 후 해당 효과(예: 타일 이동 보조, HP 회복, 스킬 쿨타임 단축 등)가 게임에 올바르게 적용되는지 확인
    - **절차:**
        1. 보상 선택 UI에서 특정 옵션 선택
        2. 선택한 효과가 즉시 또는 일정 턴 이후 발동되는지 확인
    - **예상 결과:** 선택한 효과가 의도한 대로 게임 플레이에 반영됨

### **다. 스킬 시스템 관련 테스트 케이스**

- **TC-07: 스킬 선택 및 초기화**
    
    - **목표:** 게임 시작 시 6개 스킬 중 하나를 선택할 수 있으며, 선택이 확정되는지 확인
    - **절차:**
        1. 게임 시작 화면에서 스킬 선택 UI 확인
        2. 한 가지 스킬 선택 후 확인 버튼 클릭
    - **예상 결과:** 선택한 스킬이 게임 내에서 활성화되고, 다른 스킬 선택이 불가능함
- **TC-08: 스킬 효과 발동 및 쿨타임**
    
    - **목표:** 스킬 사용 시 고유 효과가 발동되고, 지정된 쿨타임 동안 재사용이 불가능한지 확인
    - **절차:**
        1. 스킬 사용 버튼 클릭 후 효과 발동 확인
        2. 스킬 재사용 시도 전 쿨타임 진행 상태 확인
    - **예상 결과:** 스킬 효과가 정상적으로 적용되고, 쿨타임 기간 동안은 스킬 버튼이 비활성화됨
- **TC-09: 스킬 사용이 게임 상태에 미치는 영향**
    
    - **목표:** 스킬 사용 후 타일 이동, 점수 산출 등 게임의 다른 부분에 미치는 영향을 검증
    - **절차:**
        1. 스킬 사용 전후 게임 상태(타일 배열, 점수, HP 등)를 비교
        2. 효과가 예상대로 반영되었는지 확인
    - **예상 결과:** 스킬 효과가 명확하게 게임 플레이에 영향을 주어, 타일 이동이나 점수 계산 방식이 변경됨

### **라. 사용자 경험 관련 테스트 케이스 (공통)**

- **TC-10: UI 반응속도 및 애니메이션**
    
    - **목표:** 각 기능(타일 배치, 보상 선택, 스킬 사용) 시 UI가 즉각 반응하며, 애니메이션이 부드럽게 진행되는지 확인
    - **절차:**
        1. 각 기능 실행 시 UI 반응 시간 측정 및 애니메이션 품질 평가
    - **예상 결과:** 입력 후 0.5초 이내 반응, 애니메이션이 끊기거나 지연 없이 진행됨
- **TC-11: 6초 제한 타이머 정확성**
    
    - **목표:** 턴당 6초 제한 타이머가 정확하게 카운트다운되는지 확인
    - **절차:**
        1. 타일 배치 화면에서 타이머 동작 확인
        2. 6초가 정확히 경과하면 자동 방치 처리 여부 검증
    - **예상 결과:** 타이머가 정확히 6초를 카운트하며, 제한 시간 종료 후 자동으로 방치 처리됨
