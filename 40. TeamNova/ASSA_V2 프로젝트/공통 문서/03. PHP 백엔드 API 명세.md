# 3. PHP 백엔드 API 명세 v1.1

## 1. API 개요

### 1.1 기본 정보

- **Base URL**: `http://localhost:8000/api/v1`
- **프로토콜**: HTTP/HTTPS
- **데이터 형식**: JSON
- **인코딩**: UTF-8
- **인증 방식**: JWT Bearer Token

### 1.2 공통 헤더

```http
Content-Type: application/json
Accept: application/json
Authorization: Bearer {jwt_token}
```

### 1.3 HTTP 상태 코드

- `200 OK`: 성공
- `201 Created`: 생성 성공
- `400 Bad Request`: 잘못된 요청
- `401 Unauthorized`: 인증 실패
- `403 Forbidden`: 권한 부족
- `404 Not Found`: 리소스 없음
- `422 Unprocessable Entity`: 유효성 검증 실패
- `500 Internal Server Error`: 서버 오류

### 1.4 데이터 모델 참조

**모든 응답 데이터 구조**: [📄 05. 주요 데이터 모델 정의 참조](./05.%20주요%20데이터%20모델%20정의.md)

## 2. 인증 API

### 2.1 회원가입

**POST** `/auth/register`

```json

  "email": "user@example.com",
  "password": "password123!",
  "password_confirmation": "password123!",
  "nickname": "사용자닉네임"
}

// Response (201)
{
  "success": true,
  "message": "회원가입이 완료되었습니다",
  "data": {
    "user": {}, // User 모델 - 05. 데이터 모델 참조

}
```

## 3. 게시물 API

### 3.1 게시물 목록 조회

**GET** `/posts`

**Query Parameters**:

- `page` (int): 페이지 번호 (기본값: 1)
- `per_page` (int): 페이지당 항목 수 (기본값: 20, 최대: 50)
- `sort` (string): 정렬 방식 (latest, oldest, popular)

```json
// Response (200)
{
  "success": true,
  "data": [
    {} // Post 모델 배열 - 05. 데이터 모델 참조
  ],
  "meta": {} // PaginationMeta - 05. 데이터 모델 참조

```

## 9. 검색 API

### 9.1 통합 검색

**GET** `/search`

**Query Parameters**:

- `q` (string): 검색어
- `type` (string): 검색 타입 (all, users, posts, chatrooms)
- `page` (int): 페이지 번호

```json
// Response (200)
{
  "success": true,
  "data": {
    "query": "검색어",
    "users": [], // UserProfile 모델 배열
    "posts": [], // Post 모델 배열
    "chatrooms": [], // ChatRoom 모델 배열
    "total_count": {
      "users": 5,
      "posts": 10,
      "chatrooms": 2
    }
  }
}
```
